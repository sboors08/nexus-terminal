# NEXUS Terminal — отчёт по frontend foundation

## Результат

Подготовлен каркас `frontend/` без реализации Dashboard и других продуктовых экранов.

## Выбранный стек

- React 19.2.7
- TypeScript 5.9.3
- Vite 8.1.4
- React Router 8.2.0 в declarative mode
- CSS Modules + CSS Custom Properties

## Почему этот стек

- подходит для одностраничного веб-терминала;
- строгая типизация снижает риск ошибок в моделях сетапов и рыночных данных;
- Vite даёт быстрый локальный запуск и сборку;
- React Router покрывает текущую маршрутизацию без преждевременного усложнения;
- CSS-токены сохраняют утверждённую дизайн-систему без привязки к UI-фреймворку.

## Подключённые маршруты

- `/` — Dashboard
- `/scanner` — Scanner
- `/workspace` — Workspace
- `/alerts` — Alerts
- `/market-history` — Market History
- `/replay` — Replay
- `/settings` — Settings
- `/dashboard` — редирект на `/`
- `*` — 404

## Перенесённые дизайн-токены

- тёмная тема;
- Inter;
- 8 px spacing grid;
- радиусы 8–12 px;
- тонкие границы;
- LONG — зелёный;
- SHORT — красный;
- наблюдение — серый;
- подход — жёлтый;
- подтверждение — фиолетовый;
- пробой/отскок — зелёный;
- информация — голубой.

## Состояние проверки репозитория

Публичный адрес `sboors08/nexus-terminal` вернул 404. Поэтому невозможно подтвердить текущие файлы, ветку и историю коммитов без доступа к приватному репозиторию. Каркас подготовлен как добавление в папку `frontend/` и не затрагивает остальные части проекта.

## Проверки

- `npm run verify:foundation` — успешно;
- `npm run typecheck` — успешно;
- `npm run build` — успешно;
- все зарегистрированные SPA URL возвращают приложение через Vite preview.

Требование среды: Node.js 22.22.0 или новее.

- Dashboard не реализовывался;
- тестовые рыночные данные не создавались;
- Binance и другие реальные источники не подключались;
- Watchlist не добавлялся в основную навигацию;
- backend и realtime-архитектура не затрагивались.
